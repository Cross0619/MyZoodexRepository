Android Studio のセットアップ

ここでは Android Studio をインストールし、最初のプロジェクトを起動するところまでを一気に進めます。内容は公式ドキュメント（Install Android Studio や Android Studio の概要）をベースにしつつ、初心者がつまずきやすいポイントを補足しています。

1. ダウンロードとインストール

  • 公式サイトから最新の Android Studio を入手
    Google が配布するインストーラが最も安全です。Windows と macOS のいずれも同じページからダウンロードできます。
  • Windows: インストーラを実行し、デフォルト設定のまま「Next」を押して進めれば問題ありません。インストール後、必要に応じて Android SDK の追加ダウンロードが始まります。
  • macOS: ダウンロードした .dmg を開き、Android Studio を Applications フォルダへドラッグします。初回起動時に Gatekeeper が警告する場合がありますが、「開く」を選べば大丈夫です。
よくあるつまずき: インストール後の追加ダウンロード中にネットワークが不安定だと失敗することがあります。安定した回線で実施しましょう。

2. 初回起動と画面の見かた

初回起動すると、テーマ（Light/Dark）と既存設定のインポート有無を尋ねられます。迷ったらデフォルトのままで構いません。その後「Welcome to Android Studio」画面が表示され、ここから新規プロジェクトの作成や既存プロジェクトのオープンができます。

3. 新規プロジェクトを作成する

公式の「Create your first project」手順に沿って、次のように進めます。

  1. Start a new Android Studio project を選択
  2. テンプレートは Empty Activity もしくは Empty Compose Activity を選びます。Compose で UI を作るなら後者が便利です。
  3. プロジェクト名と保存先を入力し、言語は Kotlin を選択
  4. 最低 SDK は「API 24 (Android 7.0)」以上が扱いやすいです（エミュレータも用意しやすいため）。

作成後、Gradle の同期が走ります。右下に進捗バーが出るので完了まで待ちましょう。

4. エミュレータ or 実機で動かす

開発初期はエミュレータが手軽ですが、実機確認もいずれ必要になります。

  • エミュレータ: 「Device Manager」から新規 Virtual Device を追加します。Pixel シリーズ + 推奨されたシステムイメージを選んでおけば OK です。
  • 実機: Android 端末の開発者向けオプションで「USB デバッグ」を有効にし、USB ケーブルで PC に接続します。接続許可のダイアログが出たら許可してください。

どちらの場合も、実行ボタン（三角形）からデバイスを選択すればビルドとインストールが始まります。

5. 次のステップへ

環境が整ったら、次は Kotlin の基本文法を確認し、Jetpack Compose で UI を作ってみましょう。続きは「Kotlin 入門」と「Compose 入門」ページで解説します。

補足メモ

・ディスク容量は 10GB 以上あると安心です。
・エミュレータが重い場合は、RAM 2GB 程度の設定にし、x86 イメージを選びましょう。
・企業や学校の PC ではプロキシ設定が必要な場合があります。「Appearance & Behavior > System Settings > HTTP Proxy」を確認してください。
