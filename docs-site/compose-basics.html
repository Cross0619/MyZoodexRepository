Jetpack Compose の基本

Jetpack Compose は「UI をコードで宣言的に書く」ための Android 公式フレームワークです。XML のレイアウトファイルは不要になり、状態に応じて UI を組み替える発想が中心になります。

@Composable 関数

UI を描く関数には @Composable アノテーションを付けます。関数名は動詞 + 名詞で「何を描くか」が伝わるようにすると読みやすくなります。

@Composable fun Greeting(name: String) { Text(text = "こんにちは、${'$'}name") }

基本の UI 要素

  • Text: 文字列を表示します。
  • Button: クリック可能なボタン。onClick にラムダを渡します。
  • Column / Row: 縦・横方向に子要素を並べるレイアウト。
@Composable fun ButtonsSample() { Column(modifier = Modifier.padding(16.dp)) { Text("Compose は宣言的 UI") Spacer(Modifier.height(8.dp)) Button(onClick = { /* TODO: 処理 */ }) { Text("タップしてみる") } } }

Modifier の役割

Modifier は「余白を入れる」「サイズを変える」など、コンポーネントに後付けで振る舞いを加えるオブジェクトです。Compose ではレイアウトや装飾を Modifier に集約することで読みやすいコードを保ちます。

状態管理の最初の一歩

Compose では「状態 (State) が変わると UI も再描画される」仕組みです。シンプルな入力フォームなら remember { mutableStateOf() } を覚えておきましょう。

@Composable fun HelloForm() { var name by remember { mutableStateOf("") } Column(modifier = Modifier.padding(16.dp)) { OutlinedTextField( value = name, onValueChange = { name = it }, label = { Text("お名前") } ) Spacer(Modifier.height(8.dp)) Text(text = if (name.isNotBlank()) "こんにちは、${'$'}name さん" else "入力を待っています") } }

ミニデモ：入力した名前を表示

上のコードをそのまま setContent { HelloForm() } に差し替えるだけで動く簡単なデモです。状態と UI が 1 ファイルで完結するのが Compose の強みです。

補足

・従来の View + XML との違いは「状態をソースにして UI を宣言する」点。
・状態を持ちすぎると追いかけづらいので、画面単位で「誰がどの状態を持つか」を意識しましょう。
